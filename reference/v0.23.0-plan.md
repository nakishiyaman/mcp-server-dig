## リサーチ結果サマリー

### MCP SDK
- 現行 1.27.1 が最新。更新不要
- MCP仕様の新機能（Sampling, Roots, Elicitation）はクライアント側機能のためサーバー側での対応不要

### Node.js 24
- **Active LTS**（2025-10-28〜）、最新 24.14.0
- Node.js 20は2026-04-30にEOL → v0.23.0でCIに24追加し、20のEOL準備
- GitHub Actionsでは2026-03-04から`FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`がデフォルト
- プロジェクトは`execFile`+配列引数のみ使用、Node.js 24の`shell: true`非推奨の影響なし

### vitest 4.1
- 4.0→4.1は**破壊的変更なし**（minor update）
- 新機能: `aroundEach`/`aroundAll`フック、テストタグ、`--detect-async-leaks`
- `@vitest/coverage-v8` も同時に4.1.0へ更新可能

### npm package最適化
- 現在: 99.2 kB (tarball) / 500.5 kB (unpacked) / 181ファイル
- **ソースマップ除外で〜40%削減可能**（`.js.map` 101ファイル、約200 kB）
- `.d.ts` は利用者にとって有用なため維持
- 方法: `package.json` の `files` に `!build/**/*.js.map` 追加

### 新ツール実現可能性

| ツール | 概要 | 複雑度 | コード再利用 | 価値 |
|--------|------|--------|-------------|------|
| `git_contributor_network` | コントリビューター間の共同作業グラフ | 低 | 85% | 高 |
| `git_conflict_history` | マージコンフリクト頻発ファイル検出 | 中 | 70% | 高 |
| `git_survival_analysis` | コード行の生存期間（チャーン速度の時系列分析） | 中 | 80% | 中 |

#### git_contributor_network
- 既存の`analyzeContributors()`、`knowledge-map.ts`を大幅再利用
- ファイル共有ベースの共同作業検出（同一ファイルを触った著者ペア）
- 出力: 著者ペアの共有ファイル数、強度スコア、サイロ検出

#### git_conflict_history
- `git log --merges --name-only` でマージコミット内のファイル頻度を集計
- マージ回数が多いファイル = コンフリクト頻発の候補
- 制約: squash-mergeリポジトリでは信号が弱い

#### git_survival_analysis
- `git log --numstat` の時系列分析（既存`parseNumstatOutput`再利用）
- daily/weekly/monthly粒度でinsertions/deletions/生存率を算出
- 「チャーン速度の推移」として実装（純粋な行レベル生存分析は計算コスト過大）

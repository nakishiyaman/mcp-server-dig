# ADR-0002: exec()ではなくexecFile()でgit実行

- Status: Accepted
- Date: 2026-01-01

## Context

MCPサーバーはユーザー提供のパラメータ（リポジトリパス、ファイルパス、ブランチ名等）をgitコマンドに渡す。`exec()` はシェル経由でコマンドを実行するため、入力値にシェルメタ文字が含まれるとコマンドインジェクションが成立する。

## Decision

全てのgitコマンド実行に `child_process.execFile()` を使用し、引数を配列で渡す。

- `exec()` の使用を禁止する
- 引数は文字列連結ではなく配列で構築する
- `executor.ts` に集約し、直接の `execFile` 呼び出しを他のファイルに散在させない

## Consequences

- シェルインジェクション脆弱性を構造的に排除できる
- シェル固有の機能（パイプ、リダイレクト等）が使えないが、gitコマンドには不要
- 全ての引数が明示的に配列に入るため、コードレビューで安全性を確認しやすい
